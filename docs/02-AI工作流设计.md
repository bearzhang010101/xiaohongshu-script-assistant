# AI工作流设计 - MCN商单脚本生成助手

> 本工作流已通过「轻醒酸奶 x @静香」案例完整验证，详见 [03-短视频脚本与分镜.md](docs/03-短视频脚本与分镜.md)。

## 一、整体工作流架构

### 5步Agent流程

```
Brief拆解 → 博主风格分析 → 脚本生成 → 风险质检 → 飞书文档写入
   (1)         (2)          (3)        (4)         (5)
```

## 二、各环节详细设计

### Step 1: Brief拆解Agent

**功能**：解析客户brief，提取核心信息

**输入**：
- 品牌名称
- 产品信息（品类、口味、核心卖点）
- 目标人群
- 投放平台
- 合规要求

**输出**：结构化brief JSON

**核心Prompt**：
```
你是一位资深MCN策划，请拆解以下商单brief，提取核心信息并输出JSON格式：

【Brief内容】
{brief_text}

请提取以下字段：
- brand_name: 品牌名称
- product_name: 产品名称
- selling_points: 核心卖点列表（数组）
- target_audience: 目标人群描述
- platform: 投放平台
- compliance_rules: 合规红线列表（数组）
- content_tone: 内容调性要求
- video_duration: 视频时长要求

输出严格JSON格式，不要其他解释。
```

### Step 2: 博主风格分析Agent

**功能**：分析目标博主的内容风格、表达方式、常用话术

**输入**：
- 博主主页链接
- 3-5条代表作品标题/文案
- 博主简介

**输出**：风格画像JSON

**核心Prompt**：
```
你是一位小红书内容专家，请分析以下博主的内容风格，为后续脚本模仿做准备：

【博主信息】
博主昵称：{blogger_name}
博主简介：{blogger_bio}
代表作品：
{representative_works}

请分析并输出以下维度的JSON：
- persona: 人设特点（3-5个关键词）
- tone: 语言风格描述
- common_phrases: 常用话术/口头禅（数组）
- content_structure: 典型内容结构（开头钩子、中间主体、结尾方式）
- hook_types: 常用钩子类型（数组）
- scene_preference: 偏好场景（数组）
- expression_style: 表达方式特点
- brand_fit_note: 适合的植入方式建议

输出严格JSON格式。
```

### Step 3: 脚本生成Agent

**功能**：基于brief和风格画像，生成符合博主风格的短视频脚本

**输入**：
- 结构化brief
- 博主风格画像
- 产品植入要求

**输出**：完整脚本（标题、字幕、分镜、植入点）

**核心Prompt**：
```
你是一位资深短视频脚本编剧，请模仿以下博主的风格，为轻食酸奶品牌创作一条小红书短视频脚本。

【品牌Brief】
{structured_brief}

【博主风格画像】
{style_profile}

【创作要求】
1. 完全模仿博主的语言风格和表达方式，使用博主常用的话术
2. 产品植入要自然，不能硬广，要融入真实生活场景
3. 选择运动后/早餐/下午茶其中一个场景进行植入
4. 脚本时长控制在30-60秒
5. 包含：标题、字幕文案、分镜设计、产品植入点、合规提醒

【输出格式】
# 脚本标题

## 基本信息
- 时长：xx秒
- 场景：健身房→独居房间（如@静香 的练后早餐场景）
- 风格：自律vlog风，自嘲式碎碎念（如@静香）

## 字幕文案
[完整字幕内容，口语化，符合博主风格]

## 分镜设计
| 时间戳 | 景别/运镜 | 画面视觉内容 | 画面花体字/字幕文案 | 设计意图与调性把控 |
|--------|----------|-------------|-------------------|-------------------|
| 00:00-00:03 | 固定中景 | iPad时钟+快闪开场 | 标题字幕 | 钩子吸引停留 |

## 产品植入点
1. [时间点] - [植入方式描述]

## 合规风险提醒
- [检查项1]
- [检查项2]
```

### Step 4: 风险质检Agent

**功能**：检查脚本的合规性、风格匹配度、植入自然度

**输入**：生成的脚本

**输出**：质检报告 + 修改建议

**核心Prompt**：
```
你是一位MCN内容风控专家，请对以下短视频脚本进行合规质检和质量评估：

【脚本内容】
{script_content}

【合规红线】
1. 不得出现减肥、降糖、治病等功效承诺
2. 不得夸大产品效果
3. 不得使用绝对化用语（最、第一、100%等）
4. 不得涉及医疗健康功效宣称
5. 产品植入要自然，不能硬广

【检查维度】
1. 合规检查：是否触碰合规红线
2. 风格检查：是否符合博主真实表达方式
3. 植入检查：产品植入是否自然，有无硬广感
4. 逻辑检查：内容是否通顺合理
5. 平台适配：是否符合小红书平台调性

请输出质检报告JSON格式：
{
  "pass": boolean,
  "compliance_issues": [],
  "style_issues": [],
  "suggestions": [],
  "risk_level": "low/medium/high"
}
```

### Step 5: 飞书文档写入Agent

**功能**：将最终脚本自动写入飞书文档，生成可分享链接

**输入**：最终脚本内容
**输出**：飞书文档链接

**实现方式**：调用飞书开放API
- 创建文档：`POST /open-apis/docx/v1/documents`
- 写入内容：`POST /open-apis/docx/v1/documents/{document_id}/blocks/{block_id}/children`

## 三、核心Prompt库

### 1. 通用风格模仿Prompt
```
请完全模仿@静香的表达方式：
- 使用"慢慢来"、"奖励自己"、"感谢今天也没有赖床的自己"等标志性词汇
- 语气真诚，像跟闺蜜分享一样
- 多用感叹号和emoji增强情绪
- 开头要有钩子，直接抛痛点或震撼结论
- 强调"亲测好吃"、"我自己也在用"的真实感
- 结尾引导互动，"你们也试试"
```

### 2. 产品植入Prompt
```
产品植入原则：
1. 场景化：在真实生活场景中自然出现（健身包、早餐桌、下午茶）
2. 功能化：结合使用场景说明好处（运动后补充蛋白、早餐顶饱）
3. 口语化：用"我最近一直在吃"、"发现一个好东西"等自然表达
4. 不硬推：不要罗列卖点，要融入体验描述
5. 有细节：可以说口感、味道，增加真实感
```

### 3. 合规约束Prompt
```
严格遵守以下合规要求：
❌ 禁止：减肥、减脂、瘦身、减重、降糖、治病、疗效
❌ 禁止：最有效、第一、100%、绝无仅有等绝对化用语
❌ 禁止：医疗功效、保健功能宣称
✅ 可以：高蛋白、饱腹感强、低负担、0蔗糖、好吃
✅ 可以：适合早餐、运动后来一杯、下午茶替代
✅ 可以：个人口感分享、日常饮食搭配
```

## 四、可复用Skill设计

### Skill名称：mcn-script-assistant

### 适用场景
- MCN机构商单脚本快速产出
- 博主风格化内容批量生成
- 品牌种草视频脚本创作
- 内容合规性自动化质检

### 输入材料
1. 客户brief文档
2. 目标博主主页链接/代表作品
3. 产品基础信息

### 执行步骤
1. Brief结构化拆解
2. 博主风格画像分析
3. 风格化脚本生成
4. 多维度风险质检
5. 飞书文档自动输出

### 输出格式
- 完整短视频脚本（含分镜）
- 合规质检报告
- 飞书在线文档链接

### 风险检查清单
- [ ] 无减肥/降糖/治病等功效承诺
- [ ] 无绝对化用语
- [ ] 无医疗健康功效宣称
- [ ] 产品植入自然不生硬
- [ ] 符合博主真实表达方式
- [ ] 符合小红书平台规范
- [ ] 品牌信息准确无误
